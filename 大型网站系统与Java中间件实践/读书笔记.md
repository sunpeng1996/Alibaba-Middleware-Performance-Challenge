	一.读完第一章，印象深刻的是网络IO的实现方式，当我们使用Socket套接字进行网络通信开发时，有三种实现方式：<br>
	1.BIO方式：<br>
		Blocking IO,采用阻塞的方式来实现。也就是一个socket需要使用一个线程来处理。发生建立连接、读数据、写数据的操作时，都可能发生阻塞。<br>
	2.NIO方式：<br>
		Nonblocking IO,基于事件驱动的思想，采用的是Reactor(反应堆)模式。这在Java实现的服务端系统中也是采用比较多的一种方式。<br>
		相对于BIO,NIO的好处是不必为每一个socket分配一个线程，可以在一个线程中处理多个socket的工作。<br>
	3.AIO方式：<br>
		AsynchronousIO,就是异步IO。AIO采用Proactor模式。AIO进行读写的时候，只需要调用read和write方法，并且需要传入CompletionHander(动作完成的处理器),<br>
		在动作完成之后，会调用CompletionHander，当然，在不同的系统上会有一些细微的差异。<br>

	一.第二章阅读完毕，主要介绍了大型网站的架构改进，从最初在自己电脑上开发web应用程序开始讲解<br>
		①首先利用Java和单机搭建一个网站；<br>
		②单机负载出现问题，从而数据库和应用分离；<br>
		③应用服务器负载问题，从而使应用服务器从单机走向集群；<br>
			服务器变成集群后，session出现问题，解决方案有四种：<br>
			1.Session Sticky 负载均衡器根据每次请求的会话标识进行请求转发；<br>
			2.Session Replication 靠应用服务器来完成session的复制，从而使session同步；<br>
			3.Session 数据集中存储，将session数据和应用服务器分离成两层，当服务器数量多的时候，这种方式的优势明显<br>
			4.Cooike Based 把session数据放在cookie中，通过cookie传递session数据，然后在web服务器上从cookie中生成对应的session数据。<br>
		④数据库读写压力变大，读写分离；<br>
			1.采用另一个数据库作为读库；<br>
			2.搜索引擎其实就是一个读库；<br>
			3.利用缓存----加速数据的读取，分为数据缓存和页面缓存。
		⑤关系型数据库存在不足，引入分布式存储系统；
		⑥读写分离后，数据库又遇到瓶颈<br>
			1.专库专用，数据垂直拆分；垂直拆分的意思是将数据库中不同业务的数据拆分到不同的数据库中；<br>
			2.垂直拆分之后的单机遇到瓶颈，数据水平拆分:水平拆分的意思就是将一个表中的数据拆到两个数据库中。<br>
		⑦数据库问题解决后，应用面对的新挑战：<br>
			1.拆分应用：根据业务的特性将应用拆开；<br>
			2.走服务化的道路：设定服务中心提供不同的业务服务。<br>
		⑧最后简单介绍了一下消息中间件：消息中间件是在分布式系统中完成消息的发送和接受的基础软件。<br>
